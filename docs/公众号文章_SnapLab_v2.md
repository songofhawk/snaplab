## 💡 定位：你的 AI 生图“外挂”

这个工具的初衷非常单纯，就是为了解决我在玩 **AI 绘画**（Gemini / Lovart / Jimeng）时遇到的那些“小别扭”。

生成的图片基本上满意了，但有些技术细节不对，这种用图像编辑软件一下子就能解决的问题，如果让 ai 去搞，它反而半天整不明白：
*   **边距太大/尺寸不符**：生成的图还还可以，但带个大白边，需要裁一下。
*   **文件太大**：就生成个小图标，居然好几M，放在应用里还得瘦身。
*   **背景颜色不对**：想让它从亮底色，变暗色，或者透明背景，结果它把主画面给我改了。
*   **需要临时标注**：想圈出画面里的 Bug 重新跑图，还得开截图软件画个圈。
*   **提取部分**：想把生成的角色扣出来做贴纸，或者只要画面的一部分。
*   **九宫格切割**：最烦的就是 2x2 或 3x3 的 Grid 图，想拿其中一张做垫图，还要手动切。

**SnapLab** 就是为了这些场景生的。

---

## ⚡️ 极致流畅的“剪贴板”工作流

为了不打断生图的“心流”，我特意优化了剪贴板支持。你完全不需要“保存文件 -> 打开软件 -> 导入文件 -> 导出文件 -> 上传”这么繁琐。

**现在的流程是这样的：**

1.  在生图工具里 **Ctrl+C** 复制图片。
2.  在 SnapLab 页面 **Ctrl+V** 粘贴。
3.  **一顿操作**（切割、抠图、画圈）。
4.  点一下小图复制，**Ctrl+V** 粘贴回 Discord 或 WebUI 继续做垫图。

全程行云流水，不需要存任何临时文件，也不用打开臃肿的 PS。

## 🔒 纯本地，0 成本

最重要的一点：**安全且免费**。

大家应该都懂那种“传图给第三方网站”的顾虑，也不知道他们会不会拿去训练。
**SnapLab** 所有的计算——无论是切图算法，还是 SAM 模型——全都通过 WebAssembly 跑在你的浏览器里。

*   ✅ **没有服务端算力消耗**（所以我也不用收你钱）。
*   ✅ **照片不出本机**（隐私绝对安全）。
*   ✅ **断网也能用**。
